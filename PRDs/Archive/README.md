# PRDs/Archive

This directory contains all PRD and reference documents that existed
prior to March 2026.

These documents have been superseded by a restructured document
hierarchy. They are retained here for reference only and must not
be treated as current requirements or specifications.

**Current documents are in the `/PRDs` directory.**

## Why Archived

The original documents were implementation specifications, not
requirements documents. They did not define business processes,
personas, or rationale — only data structures and field definitions.

The new document hierarchy follows a four-level structure:

- **Level 1 — Master PRD** — what the CRM provides to the organization
- **Level 2 — Domain PRDs** — how each business domain operates
- **Level 3 — Consolidated Design** — authoritative entity definitions
- **Level 4 — Verification Spec** — confirms YAML matches PRDs

The archived documents served as source material for producing the
new structured PRDs. Their content has been reviewed and incorporated
where applicable.
